# First $10K/mo GTM Plan (60-day)

**Date:** 2026-03-09  
**Owner:** Tu/Jordan  
**Context:** Fast-revenue track before broadening the web control plane.

## Objective

- **Cross-reference:** see `gtm/meta/tu.md` for long-term Aiki presence/brand plan.

Generate **$10K/month run-rate within 60 days** by prioritizing paid adoption for teams already using AI coding workflows (especially clawbot-adjacent environments), then upsell to broader control-plane use.

## Core hypothesis
The fastest path is **service-first, then productized recurring usage**:

1. Deliver working outcomes quickly via a fixed-scope **Aiki FastTrack** engagement.
2. Convert delivery into recurring managed operations / platform subscriptions.
3. Expand to **General Workflow** packages only after coding value is proven.

---

## Priority offers (ranked)

### 1) Aiki FastTrack (Primary)
**Position:** Scale clawbot or AI coding assistants from “promising but fragile” to predictable production behavior.

**Offer:** 1–2 week paid sprint for 1–3 coding bots/repos.
- Repository/hook setup (provenance + review gates)
- Auto-apply safety rules (what can/can’t be applied automatically)
- Escalation flow + review handoff policy
- Baseline dashboard and weekly health report

**Pricing:**
- **One-time:** $4k–$7k (2-week sprint)
- **Optional operations retainer:** $1.5k–$3k/mo

**Why now:** Highest urgency, quickest proof, strongest close probability.

---

### 2) General Workflows Pack (Expansion, layer 2)
**Position:** Reuse the same spine for non-coding work: issue triage, QA/release workflows, support handoffs.

**Offer:** Add-on sprint or post-launch extension.
- Workflow templates + integration to issue/PM/docs/chat channels
- Team reporting and policy pack extension

**Pricing:**
- **Add-on:** $2k–$4k
- **Recurring add-on:** +$1k–$2k/mo

**Why now:** Good expansion for already onboarded teams, not first lead focus.

---

### 3) Clawbot Integration / Enablement
**Position:** Integrate a team’s internal or external bot stack with Aiki’s safety/review loop.

**Pricing:**
- $1.5k–$3k per integration
- Optional maintenance: $500–$1.5k/mo

**Why now:** Useful to sell in parallel with #1, especially to firms that already sell or consume bots.

## 60-day execution plan

### Days 1–7
- Finalize one-pager offer + 1-page scope template for **Aiki FastTrack**.
- Build a 10-target list (AI-native teams, dev agencies, infra teams adopting bots).
- Publish a one-click discovery call offer: **$5k fixed transformation sprint, 2-week delivery**.

### Days 8–21
- Launch outbound (10–15 touches/day, split across direct + referrals).
- Run 4–6 discovery calls, target 2 signed pilots.
- Hold first pilot kickoff by end of week 3.

### Days 22–35
- Deliver pilot #1 + pilot #2.
- Capture 4–6 measurable outcomes:
  - review-cycle time down
  - failed-auto-fix rate down
  - intervention load by humans down
  - fewer repeated regressions
- Convert each active pilot into support retainer before close.

### Days 36–45
- Build one de-identified case note (with metrics).
- Convert outputs into website/blog/email proof asset.
- Run follow-up wave: 20 more touches/week.

### Days 46–60
- Close 3 paid engagements across:
  - 2 Aiki FastTrack sprints + 1 retainer, **or**
  - 1 Aiki FastTrack + 1 Clawbot integration + 1 retainer
- Verify monthly run-rate and adjust pricing if needed.

## Target pipeline for $10k run-rate

Minimum viable mix:
- **2 x Aiki FastTrack @ $5k each** = $10k in first 60 days (gross),
- or **1 Aiki FastTrack + 1 Clawbot + 1 retainer** = $4k + $2.5k + $2.5k = $9k (close gap with second sprint/retainer).

## Go/no-go metrics (check at Day 30)
- Pipeline value created: **>$25k** (3x target)
- Conversion intent: **>=3 qualified conversations / week**
- Pilot start velocity: **>=1 paid pilot per 10 touches**
- Early retention signal: **pilot-to-retainer conversion >40%**

## Risks
- Scope creep on sprint packages: lock with written acceptance criteria.
- Slow response on outbound: pre-qualify by pain and tool maturity (not by budget alone).
- 5.4 model instability risk: avoid building new commitments around unstable model assumptions.
