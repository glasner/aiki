# Meta: ClawMart Strategy

**Status:** internal working draft  
**Date:** 2026-03-17

---

## Why ClawMart matters

ClawMart is not primarily a monetization surface for Aiki.

It is a:
- distribution surface,
- language-learning surface,
- product-education surface,
- and demand-generation surface.

The job of ClawMart is to help the market understand a concrete slice of Aiki value through a packaged, installable offer.

That means the first ClawMart offer should optimize for:
- adoption,
- clarity,
- reviews,
- proof,
- and pull-through into higher-value Aiki work.

Not immediate revenue.

---

## Core strategy

Aiki should use ClawMart with a two-layer strategy:

### Layer 1: Free adoption wedge
A free skill that teaches claws a better way to run coding agents.

Goal:
- drive installs,
- validate pain language,
- produce reviews,
- prove the method,
- and make the Aiki wedge legible.

### Layer 2: Premium upgrade path
A higher-touch Aiki FastTrack offer for claws.

Goal:
- install and tune the method for real repos and teams,
- customize the claw’s coding management behavior,
- and turn free interest into meaningful paid work.

So the shape is:
- **free for reach**
- **premium for depth**

---

## What we are actually selling

We are **not** primarily selling:
- workflow software,
- generic orchestration,
- or abstract SDLC theory.

We are selling a behavioral upgrade:

> the claw stops managing coding through one long fragile chat and starts managing coding agents through a more reliable method.

That is the core transformation.

For the user:
- the claw stays the interface
- the human interaction model stays familiar

Under the hood:
- the claw learns better task delegation
- the claw learns separate planning / review / fix passes
- the claw learns explicit done criteria
- the claw learns better escalation and partial-completion handling

That is the value.

---

## The first offer must be free

Recommendation:

> **The first ClawMart offering should be free, period.**

Reason:
- the main job is adoption,
- we need market language and proof,
- we need reviews,
- we need to teach the category,
- and this is the cleanest wedge into premium FastTrack work.

Charging too early would slow the main loop down.

This should be treated as:
- product marketing,
- distribution,
- and offer validation,

more than direct revenue.

---

## Language strategy

The strongest problem language is not:
- SDLC
- orchestration
- software factory
- workflow runtime

The strongest real user language is:
- long sessions drift
- long chats get fragile
- the agent loses the plot
- it says it is done when it is not
- planning, coding, and checking blur together
- partial completions become messy
- humans end up babysitting
- you need separate review / verification passes

So the ClawMart language should stay grounded in:
- drift,
- fake-done completion,
- blurred phases,
- partial completion recovery,
- less babysitting,
- and “manage coding agents like a team.”

---

## Headline strategy

The current winning concept is:

### **Give Your Claw a Coding Team**

Why it works:
- names a transformation,
- makes the claw the protagonist,
- feels bigger than “prompt better,”
- and leaves room for multi-agent behavior under the hood.

Current subtitle direction:

> *Teach your claw to manage multiple coding agents through proper stages for planning, review, fixes, and explicit done criteria.*

This is a good working direction for the free wedge.

## The free offer

### Working title
**Give Your Claw a Coding Team**

### What it is
A free skill/package that teaches claws a better default method for managing coding agents.

### What it should include
- staged coding delegation patterns
- separate plan / review / fix passes
- explicit done criteria
- escalation rules
- partial completion handling
- reusable task patterns for coding work

### What it should do commercially
- drive installs
- teach the category
- generate reviews
- create proof language
- surface the best premium upgrade conversations

---

## The premium offer

The premium should not just be “more templates.”

The premium should be a higher-touch service/install motion, e.g.:
- **Aiki FastTrack for Claws**
- **Aiki Clawbot FastTrack**

### What premium includes
- clawbot coding workflow audit
- one-shot / multi-agent control install
- repo/team-specific task templates
- review/fix loop setup
- explicit done criteria design
- escalation rules
- real-task tuning on live repos
- short async tuning period after install

### What premium sells
We are not selling content.
We are selling:
- a tuned operating method,
- installed into the claw,
- for the customer’s real repos and engineering workflow.

---

## Success criteria

### For the free offer
Success means:
- installs
- reviews
- repeated language in reviews
- people can restate the value in one sentence
- inbound asking for help applying it to real repos

### For the overall ClawMart strategy
Success means:
- the free offer creates clear pull-through into FastTrack-style work
- Aiki becomes associated with “teaching claws to manage coding agents better”
- the listing becomes a language and proof engine for broader GTM

---

## Risks

### 1) Over-abstracting the copy
If we drift into Aiki-native conceptual language, the listing will underperform.

### 2) Charging too early
If we try to monetize the wedge instead of using it for distribution, we shrink the opportunity.

### 3) Selling workflow to the human instead of behavior to the claw
The product is stronger when the claw is the protagonist.

### 4) Under-defining the premium path
If the free offer lands but the premium upgrade is vague, we lose the real business opportunity.

---

## Immediate next steps

1. Finalize the free listing copy.
2. Keep the first ClawMart offer free.
3. Create a separate premium FastTrack-for-claws doc.
4. Keep refining language using real review/problem phrasing from the market.
5. Use the free listing as the top-of-funnel wedge, not the revenue center.

---

## Bottom line

ClawMart is the right place to teach the market one concrete thing:

> **Aiki teaches claws how to manage coding agents better.**

The first offer should be free.
The business comes from what that free offer makes possible next.
