Parallel stream rendering for ratatui:

Fan-out / fan-in using box-drawing characters:
━━┬━ fan-out (one node → multiple parallel streams)
├━ middle parallel stream
╰━ last parallel stream
━━┬━ / ━━╯ fan-in (multiple streams → one node)

Parallel review shown as vertical stack:
When review checks run in parallel, they fan out vertically with individual pass/fail status next to each. This replaces the old linear ● ━━ ● ━━ ● track with a stacked view showing each stream independently.

Compressed parallel notation for --all and --short:
Full view shows the tree. Compressed view for multi-task collapses parallel streams into a single line with a parallel indicator: [2‖] means "2 streams running in parallel right now." This keeps multi-task rows scannable without drawing full DAGs for every task.

How the DAG gets built:
Each task template defines subtask dependencies as a DAG. The renderer walks the DAG and lays it out:
1. Sequential deps → horizontal ━━ connectors
2. Independent siblings → vertical ┬ ├ ╰ fan-out
3. Join points → ┬ ╯ fan-in
4. Layout uses topological sort, with parallel nodes stacked vertically at the same horizontal position

Node states:
● completed   ◉ active   ○ pending   ✓ phase passed   ✗ phase failed

Log entries show parallelism:
When multiple subtasks complete or start at the same time, they're shown on one log line separated by │ with a (parallel) suffix. This keeps the log compact without losing information about concurrent work.
