What modern coding agents actually ship:

Ghostty (Mitchell's terminal) → JetBrains Mono embedded + built-in Nerd Fonts fallback
Claude Code → inherits terminal font, uses Unicode ⏵ (broke on Linux — lesson learned)
OpenCode/Crush → BubbleTea renderer, inherits terminal, truecolor themes
ChatGPT → Segoe UI (interface) + Victor Mono (code input)
Warp → custom font based on Hack
VS Code → Consolas (Win) / Menlo (Mac) — most devs change this

The cult favorites developers actually use:
• JetBrains Mono — dominant default. Tall x-height, free, the "safe pick"
• Berkeley Mono — $75 premium, cult following. Tobi Lutke endorses. The Mitchell Hashimoto + Ghostty crowd. Can't bundle it (license prohibits terminals/IDEs)
• Monaspace — GitHub's superfamily, 5 faces, "texture healing." Ambitious
• Fira Code — ligature pioneer, still massive in VS Code
• Geist Mono — Vercel's entry. Swiss minimalism. Growing fast
• Maple Mono — rising newcomer, popular italic complement to Berkeley
• MonoLisa — premium, designer favorite

Not demoed above (not on Google Fonts): Berkeley Mono, Monaspace, Cascadia Code, MonoLisa, Maple Mono, Iosevka. These are GitHub/self-hosted only.


The real question for Aiki — it's not about the font, it's about the characters:

Aiki doesn't choose the font. The user's terminal does. What we control is which Unicode codepoints we emit. Claude Code learned this the hard way — they used ⏵ (U+23F5, Miscellaneous Technical block) and it broke on minimal Linux installs.

Safe Unicode tiers for our symbols:

Tier 1 — works everywhere: ASCII only
* + - = | [ ] # > .

Tier 2 — works in all modern coding fonts: Box Drawing + Geometric Shapes
● ○ ✓ ✗ ─ │ ┬ ├ ╯ ╭ ╰ █ ░ ▸ ▪

Tier 3 — needs a good font: Misc Symbols
◉ ◆ ▓ ━ ╌

Tier 4 — risky, avoid: Miscellaneous Technical
⏵ ⎔ ⎡ ← Claude Code's mistake

Our design uses Tier 2–3. The main risk is ◉ (U+25C9, "fisheye") for active nodes. Fallback: use ● with bright/bold color to distinguish active from completed.

Recommendation:
• Design targeting JetBrains Mono (Ghostty default, what Mitchell uses)
• Test against Fira Code + Source Code Pro as common alternatives
• Ship --ascii fallback mode for degraded environments
• Never use codepoints above U+2600 without a fallback path
• Box-drawing chars (╭╰┬├━─│) are universally safe in monospace fonts
