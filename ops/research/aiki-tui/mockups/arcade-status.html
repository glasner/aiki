Implementation notes for ratatui:

• Characters: ᗧ (Pac-Man), ᗣ (ghost), ● (eaten dot), · (pending dot), ◉ (power pellet/milestone), ║ (lane walls)
• Colors: Standard ANSI 16 — yellow (Pac-Man), blue (maze walls), red/pink/orange/cyan (ghosts map to issue severity), green (completed)
• Animation: Pac-Man mouth alternates ᗧ / ᗤ on 100ms tick. Scared ghosts alternate blue/white. Power pellet pulses.
• Score mapping: BUILD subtask = 100pts, REVIEW check = 200pts, FIX issue = 300pts, ALL CLEAR bonus = 1000pts
• Ghost → Issue mapping: BLINKY (red) = errors, PINKY (pink) = security, CLYDE (orange) = warnings, INKY (cyan) = info
• Retries as lives: Each ᗧ in the lives row = one remaining retry attempt. Dimmed = used.
• The metaphor holds: Building = eating dots. Issues = ghosts blocking path. Fix task = power pellet that lets you eat ghosts.
