Tecmo Bowl → Aiki mapping for ratatui implementation:

Characters: ☻ (agent/player), ⛨ (defender/issue), ◈ (completed subtask), ★ (touchdown/commit), ⚑ (timeout/retry)

The field = the pipeline:
• 0–40 yard line = BUILD phase (subtasks as yard markers)
• 40–80 yard line = REVIEW phase
• 80–end zone = FIX phase
• Touchdown = clean commit to Git

Football → Dev mapping:
• Quarters = iteration number (1st QTR = first attempt, 2nd = after review, etc.)
• Down & distance = current subtask progress within a phase
• Sack = review finds issues (agent stopped in their tracks)
• Tackle broken = issue fixed (agent breaks through a defender)
• Punt = escalate to human (giving up possession of the problem)
• Hurdle = ignore warning and commit anyway
• Timeouts = retries remaining
• Drive summary = provenance record

Multi-game view = the "Sunday Scoreboard" showing all concurrent tasks as simultaneous games. Each game is a compressed single-line field view.

Play call screen = the action selection when review fails. Mirrors the original Tecmo Bowl play selection UI with the cursor highlighting options.

Scoreboard = ratatui Block widget with the classic AGENT vs ISSUES layout, quarter/iteration, and game clock (elapsed time).
