Visual vocabulary for ratatui:

Track markers:
● completed step    ◉ active step    ○ pending step    ━━ connector    ▪ issue/blocker

Status icons:
✓ passed/done    ✗ failed    ▸ in progress    ◆ event marker    — not started

Colors (muted, not neon):
• green (#5fcc68) — completed, passing, build phase
• cyan (#5bb8c9) — review phase, informational
• red (#e05555) — errors, failures
• magenta (#c470b0) — security issues
• orange (#cc8844) — warnings, agent name (Cursor)
• yellow (#d4a840) — active/in-progress
• dim (#3a3a44) — inactive, pending, chrome

Three density levels:
• aiki status — focused single-task view with expanded lanes, subtask list, log
• aiki status --all — multi-task board, compressed lanes per task, shared log
• aiki status --short — one line per task, all three lanes inline, no log

The fix lane shows issues as blockers on the track:
● ━━ ◉ ━━ ○ ╌╌╌╌ ▪ ╌╌ ▪
Completed fixes are solid dots. The dashed section (╌╌) between fixes and issues visually separates "work done" from "work remaining." Issue markers (▪) sit on the track as obstacles to clear. When fixed, they become regular dots.

ratatui widgets:
• Lanes → custom widget using Span sequences in a Line
• Box → Block with Borders::ALL and rounded corners
• Log → List widget with styled items
• Multi-task → Layout with vertical splits per task
• Short mode → just raw Paragraph lines, no widgets needed
