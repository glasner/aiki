plans in aiki status

Three options for integrating plan files into the pipeline model
OPTION A — Plans as sidebar/header

Plans live in a left panel or top section. Tasks show in the main area as pipeline tracks. The plan list is a navigation element — select a plan to filter tasks, or see which plans have no tasks yet. Feels like a file tree + main view split (lazygit, k9s).

OPTION B — Plan as "phase 0" on the track

The pipeline extends leftward: plan → build → review → fix. The plan phase shows whether the plan is being actively written (interactive session), is ready, or hasn't started. Every task's track begins with its plan origin. Consistent pipeline metaphor — no separate UI element.

OPTION C — Separate views with tab switching

Two modes: [P] Plans and [T] Tasks. Plans view is a simple list of markdown files with status indicators. Tasks view is the full pipeline monitor. Switch between them with a keypress. Clean separation — each view does one thing well.

Comparison summary:

Option A — Sidebar
+ Plans always visible — easy to see what's idle, what's active
+ Natural [e] edit and [b] build targets from plan list
+ Familiar split-pane pattern (lazygit, k9s)
- Sidebar eats ~20 columns of width — tighter pipeline area
- Two navigation contexts (plan pane vs task pane) — needs Tab switching or h/l
- At Tier 3/4 scale the sidebar becomes noise — 50 plans in a sidebar doesn't help

Option B — Phase 0 on track
+ Consistent metaphor — everything is a pipeline phase
+ No extra UI element — scales naturally with density tiers
+ Plan-in-progress is just another active node on the track
- Adds a column to every row at Tier 2+ even when plan is done (just a ●)
- Tasks without plan origin (ad-hoc tasks) have an empty plan phase — awkward
- The "planning" state is very different from build/review/fix — feels forced as a pipeline phase

Option C — Separate tabs
+ Each view is clean — plans view shows plan-level info, tasks view shows pipelines
+ Full width for both views
+ Tabs scale independently — plan view doesn't get crowded at Tier 4
+ Plan view can show plan-planific info (task tree under each plan, last activity)
- Plans not visible while watching tasks — have to switch tabs to see idle plans
- Two views to maintain in the implementation
- Context switching between tabs — "which plan is this task from?" requires a tab switch
